# プロジェクト仕様書: データ活用・プライバシー炎上事例カタログ

## 目的

データ活用やプライバシーに関する炎上・トラブル事例を体系的に収集・整理し、企業のデータガバナンス改善に役立てるWebカタログを構築する。

## スコープ

- データ漏洩、プライバシー侵害、同意なし利用、不適切な匿名化、アルゴリズム差別、監視・追跡に関する国内外の事例を対象
- 公開情報（ニュース記事、規制当局の発表、学術論文等）に基づく事例のみ収録
- 未確認情報や推測に基づく事例は収録しない

## データモデル

### case.json

| フィールド | 型 | 必須 | 説明 |
|-----------|---|------|------|
| id | string | ○ | 事例ID（例: breach-0001） |
| title | string | ○ | 事例タイトル（30〜60文字） |
| region | string | ○ | 国内 / 国外 |
| domain | string | ○ | 業界ドメイン |
| organization | string | ○ | 関連組織名 |
| incident_category | string[] | ○ | 事例カテゴリ（複数選択可） |
| severity | string | ○ | 深刻度（critical/high/medium/low） |
| occurred_at | string \| null | ○ | 発生日（YYYY / YYYY-MM / YYYY-MM-DD / null） |
| summary | string | ○ | 何が起こったか（事件の経緯） |
| impact | string | ○ | どのような影響・被害があったか |
| root_cause | string | ○ | なぜ起こったか（原因分析） |
| response | string | ○ | どう対応したか（企業・規制当局の対応） |
| lessons_learned | string | ○ | 何を学べるか（教訓・再発防止策） |
| tags | string[] | ○ | 検索用タグ |
| sources | Source[] | ○ | 出典（最低1件必須） |
| figures | Figure[] | ○ | 図表 |
| review_status | string | ○ | レビュー状態 |

### 事例カテゴリ

| カテゴリ | 説明 | 例 |
|---------|------|-----|
| data_breach | データ漏洩 | 不正アクセス、内部犯行、設定ミスによるデータ流出 |
| privacy_violation | プライバシー侵害 | 個人情報の不正利用、過度な収集 |
| unauthorized_use | 同意なし利用 | 通知なしのデータ利用目的変更、第三者提供 |
| inadequate_anonymization | 不適切な匿名化 | 再識別リスク、匿名化の不備 |
| algorithmic_discrimination | アルゴリズム差別 | AIによる差別的判断、バイアス |
| surveillance_tracking | 監視・追跡 | 過度な監視、位置追跡 |

### 深刻度（severity）判定基準

| 深刻度 | 基準 |
|--------|------|
| critical | 大規模なデータ漏洩（100万件以上）、重大な人権侵害、巨額の制裁金 |
| high | 大規模な影響（数万〜数十万件）、規制当局による制裁、広範な社会的批判 |
| medium | 一定の影響、企業の信頼低下、改善命令 |
| low | 限定的な影響、自主的な改善で解決 |

## ユースケース

1. **一覧表示**: 事例を一覧で表示し、カテゴリ・ドメイン・深刻度でフィルタリング
2. **検索**: キーワードで事例を全文検索
3. **詳細表示**: 事例の詳細情報、出典、図表を表示
4. **事例追加・編集**: UIまたはCLIで新規事例を追加、既存事例を編集
5. **統計表示**: カテゴリ別・ドメイン別・年別の統計グラフ

## 非機能要件

- GitHub Pages による静的サイトホスティング
- レスポンシブデザイン（モバイル対応）
- 全データをクライアントサイドで保持（サーバー不要）
